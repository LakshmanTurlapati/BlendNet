<objective>
Solve the cross-compilation chicken-and-egg problem and produce the missing blender.wasm binary.

Purpose: Plan 04 identified the root blocker: Blender's CMake builds host tools (makesdna, makesrna, datatoc, shader_tool) as WASM executables via emcmake, but these must run natively during the build to generate source files (DNA structs, RNA definitions, datatoc resources, shader metadata). The fix is a two-pass approach: (1) build all four host tools natively with GCC-14, (2) patch the generated build.ninja to replace WASM tool references with native executables, (3) run the WASM compilation. This is the single remaining blocker preventing Phase 1 completion.

Output: build-wasm/blender.wasm, build-wasm/blender.wasm.br, build-wasm/blender.js, plus reusable build scripts
</objective>

    Create three files that together solve the cross-compilation host tool problem:

    **File 1: scripts/patch-ninja-host-tools.sh**

    A script that patches the generated build.ninja to replace WASM host tool references with
    native executables. This is the KEY fix for the chicken-and-egg problem. The script must:

    1. Accept two arguments: BUILD_DIR (path to build-wasm/) and NATIVE_DIR (path to build-native/)
    2. In build.ninja, perform these replacements:
       a. Replace `bin/makesdna.js` with the native makesdna path in all CUSTOM_COMMAND lines
          - The COMMAND lines look like: `COMMAND = cd ... && /usr/bin/cmake -E env ... /src/build-wasm/bin/makesdna.js <args>`
          - Replace `/src/build-wasm/bin/makesdna.js` with `/src/build-native/makesdna`
       b. Replace `bin/makesrna.js` with native makesrna path in COMMAND lines
          - Replace `/src/build-wasm/bin/makesrna.js` with `/src/build-native/makesrna`
       c. Replace `bin/datatoc.js` with native datatoc path in COMMAND lines
          - Replace `/src/build-wasm/bin/datatoc.js` with `/src/build-native/datatoc`
       d. Replace `bin/shader_tool.js` with native shader_tool path in COMMAND lines (if shader_tool
          custom commands exist)
    3. For the "build bin/X.js: CXX_EXECUTABLE_LINKER__X_Release ..." link targets, replace them
       with phony rules that point to the native executables. This prevents ninja from trying to
       link the WASM versions. The approach:
       - Comment out (or replace) the `build bin/makesdna.js: CXX_EXECUTABLE_LINKER__makesdna_Release ...` line
       - Add: `build bin/makesdna.js: phony /src/build-native/makesdna`
       - Do the same for makesrna, datatoc, shader_tool
    4. Use sed for the replacements. The script must be idempotent (safe to run multiple times).
    5. Validate: after patching, grep for remaining `.js` references to host tools in COMMAND lines
       and report any that were missed.

    The exact sed commands needed (these are the core of the fix):

    ```bash
    # Replace COMMAND references to WASM host tools with native ones
    sed -i "s|/src/build-wasm/bin/makesdna\.js|${NATIVE_DIR}/makesdna|g" "${BUILD_DIR}/build.ninja"
    sed -i "s|/src/build-wasm/bin/makesrna\.js|${NATIVE_DIR}/makesrna|g" "${BUILD_DIR}/build.ninja"
    sed -i "s|/src/build-wasm/bin/datatoc\.js|${NATIVE_DIR}/datatoc|g" "${BUILD_DIR}/build.ninja"
    sed -i "s|/src/build-wasm/bin/shader_tool\.js|${NATIVE_DIR}/shader_tool|g" "${BUILD_DIR}/build.ninja"

    # Replace linker rules with phony rules pointing to native executables
    sed -i '/^build bin\/makesdna\.js: CXX_EXECUTABLE_LINKER/c\build bin/makesdna.js: phony '"${NATIVE_DIR}"'/makesdna' "${BUILD_DIR}/build.ninja"
    sed -i '/^build bin\/makesrna\.js: CXX_EXECUTABLE_LINKER/c\build bin/makesrna.js: phony '"${NATIVE_DIR}"'/makesrna' "${BUILD_DIR}/build.ninja"
    sed -i '/^build bin\/datatoc\.js: CXX_EXECUTABLE_LINKER/c\build bin/datatoc.js: phony '"${NATIVE_DIR}"'/datatoc' "${BUILD_DIR}/build.ninja"
    sed -i '/^build bin\/shader_tool\.js: CXX_EXECUTABLE_LINKER/c\build bin/shader_tool.js: phony '"${NATIVE_DIR}"'/shader_tool' "${BUILD_DIR}/build.ninja"
    ```

    IMPORTANT: The `|| bin/makesdna.js` order-only dependency references in other build rules also
    need to work. Since we are making `bin/makesdna.js` a phony target that depends on the native
    executable, ninja will consider it "built" when the native executable exists. This is correct.

    first attempt if it seems too complex. makesrna depends on essentially ALL of Blender's data

    - For makesrna, try building it natively. If it fails due to excessive dependencies, use the
      fallback: since CMAKE_CROSSCOMPILING_EMULATOR=node is set, the WASM makesrna.js CAN be
      executed via node. In that case, do NOT patch the makesrna COMMAND lines in build.ninja --
      only patch makesdna, datatoc, and shader_tool. CMake will prepend the emulator ("node") to
      the executable path automatically for custom commands.

    Actually, upon further analysis: Emscripten's emcmake sets CMAKE_CROSSCOMPILING=TRUE and
    CMAKE_CROSSCOMPILING_EMULATOR=node. This means CMake's add_custom_command with $<TARGET_FILE:makesrna>
    should ALREADY prepend "node" to the command. Let me verify this by checking the COMMAND line
    in build.ninja. From line 66253:
    `COMMAND = cd ... && /usr/bin/cmake -E env _DUMMY_ENV_VAR_=1 /src/build-wasm/bin/makesrna.js ...`

    It does NOT prepend node -- it just uses the .js path directly. This means
    CMAKE_CROSSCOMPILING_EMULATOR is not being applied to custom commands (likely because Blender
    uses add_custom_command with a direct target file reference, not through the emulator).

    So the ninja patch approach IS necessary for all four tools. But building makesrna natively
    is very hard. The ALTERNATIVE for makesrna:
    - In the patch script, replace the makesrna COMMAND to prepend "node" before the .js path
    - i.e., change `COMMAND = ... /src/build-wasm/bin/makesrna.js <args>` to
      `COMMAND = ... node /src/build-wasm/bin/makesrna.js <args>`
    - This requires that makesrna.js was successfully linked by emcc (which may be the issue that
      causes the build to fail). If it links, we can run it via node.

    DECISION: Use this hybrid approach:
    - Build makesdna, datatoc, shader_tool natively with GCC-14 (they are small/standalone)
    - For makesrna: let it link as WASM (bin/makesrna.js), then patch COMMAND to run via "node"
    - Patch build.ninja:
      - makesdna, datatoc, shader_tool: replace with native executables
      - makesrna: prepend "node" to the COMMAND if not already there
    - This avoids the need to build the complex makesrna natively

    Update cmake/host_tools/CMakeLists.txt to add shader_tool only (makesdna and datatoc are

<objective>
Create the custom headless WASM entry point, compile Blender to WebAssembly, compress the output with Brotli, and validate threading/memory/main loop behavior.

Purpose: This is the core deliverable of Phase 1 -- an actual running WASM binary that proves Blender's engine compiles and initializes in a browser context. The custom entry point bypasses Blender's window manager (no WM_main) and provides exported C functions that JavaScript can call to initialize the engine, load .blend files, query scene data, and check memory usage. Post-build compression (BUILD-07) ensures the binary is served efficiently.

Output: Compiled blender.wasm binary with exported functions, Brotli-compressed .wasm.br file, validated via test scripts from Plan 02.
</objective>

       ```cpp
       #include "BKE_blendfile.hh"
       #include "BKE_main.hh"

       #include <cstring>
       ```

       Implement exported functions (all wrapped in extern "C" for C linkage):

       a. `EMSCRIPTEN_KEEPALIVE int wasm_init()`:
          - Call CLG_init() for logging
          - Call BKE_appdir_init() for directory setup
          - Initialize MEM_guarded_alloc (it auto-initializes, but call MEM_init_memleak_detection if available)
          - Initialize DNA via BKE_blender_globals_init() or minimal equivalent
          - Print "[wasm] Blender WASM engine initialized" to stdout
          - Return 0 on success, -1 on failure

       b. `EMSCRIPTEN_KEEPALIVE int wasm_load_blend(const char *path)`:
          - Use BlendFileReadReport for error reporting
          - Call BKE_blendfile_read() or BLO_read_from_file() to load the .blend file from Emscripten MEMFS
          - If load fails, return -1
          - Count objects in the scene: iterate G_MAIN->objects via LISTBASE_FOREACH
          - Print "[wasm] Loaded N objects from path"
          - Return number of objects

       c. `EMSCRIPTEN_KEEPALIVE const char* wasm_query_scene()`:
          - Build a simple JSON string (use a static buffer, 4KB max) containing:
            - scene name
            - object count
            - array of object names and types
          - Return pointer to the static buffer
          - Format: {"scene":"Scene","objects":[{"name":"Cube","type":"MESH"},...]}

       d. `EMSCRIPTEN_KEEPALIVE size_t wasm_memory_usage()`:
          - Return MEM_get_memory_in_use() (from guardedalloc)

       e. `int main(int argc, char **argv)`:
          - Call wasm_init()
          - Print "[wasm] Headless WASM build ready. Awaiting commands."
          - If argc > 1 and argv[1] is a .blend path, call wasm_load_blend(argv[1])
          - Return 0 (do NOT enter WM_main -- headless mode)

       NOTE: This is a MINIMAL entry point. It bypasses Blender's full initialization (no Python, no GPU, no window manager). The goal is to prove the data pipeline works: DNA -> RNA -> BKE -> blenloader.

       If certain BKE functions are not available due to disabled features, use #ifdef guards or stub them. The key test is: can we initialize MEM_guarded_alloc, load DNA types, and read a .blend file.

       At "# === SECTION: Post-Build ===" add Brotli compression step (BUILD-07):
       ```bash
       # === SECTION: Post-Build ===
       # Brotli compression for serving (BUILD-07)
       if [ -f build-wasm/blender.wasm ]; then
           WASM_SIZE=$(stat -f%z build-wasm/blender.wasm 2>/dev/null || stat -c%s build-wasm/blender.wasm 2>/dev/null)
           echo "[build] Uncompressed WASM size: ${WASM_SIZE} bytes ($(echo "scale=2; ${WASM_SIZE}/1048576" | bc) MB)"

           echo "[build] Compressing with Brotli (--best)..."
           brotli --best --force build-wasm/blender.wasm -o build-wasm/blender.wasm.br

           BR_SIZE=$(stat -f%z build-wasm/blender.wasm.br 2>/dev/null || stat -c%s build-wasm/blender.wasm.br 2>/dev/null)
           echo "[build] Brotli compressed size: ${BR_SIZE} bytes ($(echo "scale=2; ${BR_SIZE}/1048576" | bc) MB)"
           RATIO=$(echo "scale=1; ${BR_SIZE}*100/${WASM_SIZE}" | bc)
           echo "[build] Compression ratio: ${RATIO}%"

           # BUILD-07 target: compressed size under 30MB
           BR_MB=$(echo "scale=0; ${BR_SIZE}/1048576" | bc)
           if [ "${BR_MB}" -gt 30 ]; then
               echo "[build] WARNING: Compressed size exceeds 30MB target (${BR_MB} MB)"
           else
               echo "[build] PASS: Compressed size within 30MB target (${BR_MB} MB)"
           fi
       else
           echo "[build] WARNING: blender.wasm not found -- skipping compression"
       fi
       ```
